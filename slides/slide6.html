AngularJS 특징

Angular의 철학[2]

  • DOM 제어와 비지니스 로직은 분리되어야한다.
  • 어플리케이션의 Client Side와 Server Side는 분리되어야 한다.
  • Angular를 통해 어플리케이션 개발의 전체 과정 UI, 비지니스 로직, 테스팅을 가능하도록 한다.

Angular의 특징[5]

  • MVC 구조를 제공
  • View와 Model의 양방향 데이터 바인딩을 지원
  • 콜백함수로부터의 해방
  • 재사용이 쉬운 UI컴포넌트 생성 가능(선언적 HTML)
  • 뷰 코드와 로직코드의 분리
  • 데이터 중심적 개발

Backbone.js와의 비교[2]

  • View와 Model의 동기화 방법 차이
  • Backbone.js는 underscore의 template() 함수를 사용하여 Templating을 구현하고 angular는 커스터마이징된 html 태그와 Angular Expression 을 이용
  • 그 외에 SPA(Single Page Application) 개발을 위한 대부분의 기능은 비슷
